# Ecommerce Data Warehouse

一个基于 Hive 的电商离线数据仓库项目，构建 ODS → DWD → DWS → ADS 分层模型，实现电商核心指标分析（GMV、商品排行、用户转化率）。

---

## 项目简介

本项目围绕电商业务数据（用户、商品、订单、支付、行为日志），构建完整离线数仓链路：

MySQL → ODS → DWD → DWS → ADS

通过数据建模与 ETL 处理，实现从原始数据到分析指标的逐层加工。

---

## 技术栈

MySQL / Hive / Hadoop
DataX / SQL / Shell

## 系统架构

### 离线数仓链路

MySQL → ODS → DWD → DWS → ADS

                              │

                  │ Hive SQL（清洗转换）
                  ▼
        ┌────────────────────────────────────┐
        │ DWD（明细层）                      │
        │ - 维度表（商品/类目）              │
        │ - 事实表（订单/支付/行为）         │
        └─────────┬──────────────────────────┘
                  │ 拉链处理（SCD2）
                  ▼
        ┌────────────────────────────────────┐
        │ DWD（拉链表）                      │

        └─────────┬──────────────────────────┘
                  │ 宽表构建
                  ▼
        ┌────────────────────────────────────┐
        │ DWD（宽表）                        │
        │ - 订单明细宽表                     │
        │ - 统一指标口径                     │
        └─────────┬──────────────────────────┘
                  │ 按主题聚合
                  ▼
        ┌────────────────────────────────────┐
        │ DWS（汇总层）                      │
        │ - 商品主题                         │
        │ - 用户主题                         │
        │ - 类目主题                         │
        │ - 行为主题                         │
        └─────────┬──────────────────────────┘
                  │ 指标计算
                  ▼
        ┌────────────────────────────────────┐
        │ ADS（应用层）                      │
        │ - GMV                              │
        │ - TopN                             │
        │ - 转化漏斗                         │
        └─────────┬──────────────────────────┘
                  ▼
            BI / 可视化 / 报表

* ODS：原始数据层（按 `dt` 分区存储）
* DWD：明细层（数据清洗、维度补充、宽表与拉链表）
* DWS：汇总层（按业务主题进行日粒度聚合）
* ADS：指标层（输出最终分析结果）

用于支持 T+1 模式的离线数据分析。

## 关键设计

### 数据仓库分层

ODS → DWD → DWS → ADS

* 分离原始数据与分析数据
* 避免重复计算，提高指标复用性

---

### 订单明细宽表

在 DWD 层构建 `dwd_order_detail_wide`：

* 关联订单、用户、商品、类目
* 统一分析口径
* 预计算明细支付金额

减少下游多表 Join，提高查询效率。

---

### 用户拉链表（SCD2）

构建 `dim_user_info_zip`：

* 记录用户属性历史变化
* 支持历史状态查询
* 使用 start_date / end_date 管理版本

---

### 用户行为转化分析

基于用户行为日志统计各阶段用户数（view、cart、order、pay），

并在 ADS 层计算转化率（浏览→加购、加购→下单、下单→支付、整体转化率），

用于分析整体转化情况。

## 快速开始

### 1. 初始化 MySQL

```bash
bash scripts/init_mysql.sh
```

---

### 2. 生成模拟数据

运行 Java 程序：

```bash
Main.java
```

---

### 3. 执行离线 ETL 总流程

执行离线 ETL 调度脚本：

```bash
bash scripts/run_daily_etl.sh 2025-10-01 2025-10-07
```

或指定入仓起始分区日期：

```bash
bash scripts/run_daily_etl.sh 2025-10-01 2025-10-07 2026-04-03
```

该脚本会按顺序完成：

* ODS 层数据导入
* DWD 基础表处理
* 用户拉链表更新
* 订单宽表生成
* DWS 层汇总
* ADS 层指标计算

---

## 调度与执行说明

项目通过 `run_daily_etl.sh` 实现离线 ETL 全流程调度，具备以下特性：

* 支持按业务日期范围批量处理数据
* 支持自定义入仓分区日期（load_dt），实现业务日期与分区解耦
* 各层（ODS / DWD / DWS / ADS）按顺序执行，保证数据依赖正确
* 脚本按阶段拆分（base / zip / wide），便于单独重跑与问题排查

脚本基于分区覆盖（INSERT OVERWRITE），保证幂等性，适用于历史数据回溯与分区级重跑场景。
